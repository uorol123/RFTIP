

  对每个匹配组（同一时刻多站观测）:
      1. 计算几何中心（质心）
      2. RANSAC 迭代:
          - 随机选2个站作为内点假设
          - 用这2个站计算候选误差 Δazimuth, Δrange
          - 用候选误差预测所有站的观测位置
          - 计算残差，判定内外点
          - 记录内点数量
      3. 选择内点最多的那次迭代结果
      4. 统计每个站被判定为外点的比例

  外点比例 > 阈值(50%) 的站 → 判定为故障站

  2. 关键改动

  - 用 误差模型（Δazimuth, Δrange）做 RANSAC，而不是线性回归
  - 正确利用 RANSAC 的筛选结果：只用健康站计算最终误差
  - 输出 fault_stations 和 outlier_rates 反映 RANSAC 的真实结果

  3. 与梯度下降的区别

  - 梯度下降：所有站一起优化，互相影响
  - RANSAC：先找出哪些站是健康的，再用健康站计算误差，结果更可靠

问题是： 随机选 2 个站，选到 2 个都是健康站的概率并不高

应该是最少2个，健康站的偏差肯定小 0.0几，然后误差站的偏差
0.2几，数据之间肯定差距很大，把误差按小到大排序。然后看差值。比如前
  n个的差值都接近，但是第n个和第n+1的差值突然变大，那就代表第n+1个的站开始到最后一个都是误差站，前n个都是健康站了。



对每个匹配组：
      1. 计算几何中心（所有站观测的平均位置）

      2. 计算每个站与中心的偏差，按偏差从小到大排序：
         排序后：[0.001, 0.002, 0.003, 0.02, 0.25, 0.3]

      3. 找跳跃点（差值突变）：
         - 计算相邻偏差的差值
         - 找到最大差值的位置（比如 0.003 和 0.02 之间）
         - 差值突变点之前是健康站，之后是故障站

      4. 结果：
         - 前 n 个是健康站
         - 第 n+1 个往后是故障站

      5. 统计所有匹配组中每个站被判定为故障站的次数

      6. 故障次数/总匹配组数 > 阈值 → 判定为故障站

  关键点：
  - 健康站偏差极小（接近 0）
  - 故障站偏差突然变大
  - 差值突变点自动区分健康站和故障站
