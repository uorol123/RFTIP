# 基于梯度下降的迭代寻优算法

## 一、算法概述

### 1.1 算法名称

**中文名称**：基于梯度下降的迭代寻优算法

**英文名称**：Gradient Descent-based Iterative Optimization Algorithm

**应用领域**：多雷达系统误差估计（Multi-Radar System Error Estimation）

### 1.2 算法简介

该算法用于解决多雷达站协同观测中的系统误差估计问题。其核心思想是：当多个雷达站同时观测同一目标时，如果各雷达没有系统误差，观测点应该在空间上重合。通过最小化匹配点空间方差来估计各雷达站的系统误差（方位角误差、距离误差、俯仰角误差）。

### 1.3 算法特点

- **无需真实值**：不需要外部真值参考，利用多雷达观测一致性
- **分步优化**：采用坐标下降法，依次优化方位角、距离、俯仰角误差
- **多步长策略**：先粗调后精调，加速收敛
- **代价函数约束**：结合空间方差与误差正则化，避免极端解

## 二、问题定义

### 2.1 系统误差模型

每个雷达站的观测误差可建模为：

$$
\begin{cases}
\Delta az_i & \text{方位角误差（度）} \\
\Delta r_i & \text{距离误差（米）} \\
\Delta el_i & \text{俯仰角误差（度）}
\end{cases}
$$

其中 $i$ 为雷达站编号。

### 2.2 观测修正公式

给定雷达站位置 $(lon_{r_i}, lat_{r_i}, alt_{r_i})$ 和原始观测 $(lon, lat, alt)$：

```
# 计算原始观测向量
az_orig, _, dist_orig = geod.inv(lon_r, lat_r, lon, lat)

# 应用误差修正
az_corr = az_orig - Δaz_i
dist_corr = dist_orig - Δr_i

# 计算修正后位置
lon_corr, lat_corr, _ = geod.fwd(lon_r, lat_r, az_corr, dist_corr)
```

## 三、算法流程

### 3.1 整体流程图

```
┌─────────────────────────────────────────────────────────────┐
│                    输入：多雷达观测数据                        │
└────────────────────┬────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────────┐
│              步骤1：航迹匹配（形成匹配组）                      │
│         同一时刻、不同雷达的观测点组成匹配组                   │
└────────────────────┬────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────────┐
│            步骤2：初始化所有误差为 0                           │
│        az_errors = {所有雷达: 0°}, range_errors = {0m}        │
└────────────────────┬────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────────┐
│         步骤3：优化方位角误差（距离、俯仰角固定为0）              │
│    使用梯度下降 + 多步长策略: [0.1°, 0.01°]                    │
└────────────────────┬────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────────┐
│       步骤4：优化距离误差（方位角固定，俯仰角固定为0）             │
│    使用梯度下降 + 多步长策略: [1000, 500, 200, 100, 50, 20]m   │
└────────────────────┬────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────────┐
│        步骤5：优化俯仰角误差（方位角、距离固定）                   │
│    使用梯度下降 + 多步长策略: [0.1°, 0.01°]                    │
└────────────────────┬────────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────────┐
│                    输出：各雷达系统误差估计                      │
│        {雷达站ID: (Δaz, Δr, Δel)}                            │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 匹配组（Match Group）

**定义**：同一时刻，多个不同雷达站对同一目标的观测集合

**示例**：
```python
match_group = [
    (1001, 1, 3600, 116.123, 39.456, 5000),  # 雷达1001观测
    (1002, 2, 3600, 116.128, 39.458, 5020),  # 雷达1002观测
    (1003, 3, 3600, 116.125, 39.454, 4995),  # 雷达1003观测
]
# 格式：(站号, 批号, 时间秒, 经度, 纬度, 高度)
```

## 四、代价函数设计

### 4.1 代价函数公式

$$
J = w_1 \cdot \sigma_{space} + w_2 \cdot \frac{1}{N}\sum_{i=1}^{N}\Delta az_i^2 + w_3 \cdot \frac{1}{N}\sum_{i=1}^{N}\Delta r_i^2 + w_4 \cdot \frac{1}{N}\sum_{i=1}^{N}\Delta el_i^2
$$

其中：
- $\sigma_{space}$：修正后匹配点的空间标准差
- $w_1, w_2, w_3, w_4$：各权重系数

### 4.2 默认权重配置

| 权重参数 | 默认值 | 说明 |
|----------|--------|------|
| $w_1$ (方差权重) | 100.0 | 使修正后点尽量集中 |
| $w_2$ (方位角权重) | 0.15 | 防止方位角误差过大 |
| $w_3$ (距离权重) | 6×10⁻⁷ | 防止距离误差过大 |
| $w_4$ (俯仰角权重) | 0.10 | 防止俯仰角误差过大 |

### 4.3 为什么需要误差平方项？

**问题**：如果只最小化方差，算法会收敛到极端解：
```
极端解：所有雷达误差 = 1000°
→ 修正后的点都跑到地球另一端重合
→ 方差 = 0，但这是错误的！
```

**解决方案**：加入误差平方项作为正则化约束，既要求方差小，又要求误差本身不能太大。

## 五、优化算法详解

### 5.1 梯度下降（坐标下降）

由于同时优化多个维度困难，采用**坐标下降法**：每次只优化一类误差，固定其他误差。

```python
# 步骤1：固定距离=0，俯仰角=0，只优化方位角
optimized_az = optimize_azimuth_errors(
    match_groups,
    initial_errors={all: 0},
    range_errors={all: 0}
)

# 步骤2：固定方位角=上一步结果，俯仰角=0，只优化距离
optimized_r = optimize_range_errors(
    match_groups,
    azimuth_errors=optimized_az,
    initial_range_errors={all: 0}
)

# 步骤3：固定方位角和距离，只优化俯仰角
optimized_el = optimize_elevation_errors(
    match_groups,
    azimuth_errors=optimized_az,
    range_errors=optimized_r,
    initial_elevation_errors={all: 0}
)
```

### 5.2 多步长策略

**目的**：平衡收敛速度和精度

**步长序列**：
- 方位角/俯仰角：`[0.1°, 0.01°]`
- 距离：`[1000, 800, 500, 200, 100, 50, 20]` 米

**优化逻辑**：
```python
for step in step_sequence:  # 递减步长
    while improved:
        for station in all_stations:
            for delta in [+step, -step]:  # 尝试正负方向
                temp_errors[station] += delta
                cost = calculate_cost(temp_errors)
                if cost < best_cost:
                    best_errors = temp_errors
                    improved = True
        if not improved:
            break  # 当前步长收敛，进入下一更小步长
```

## 六、算法示例

### 6.1 简化示例

假设3个雷达站同时观测一个目标：

| 雷达 | 位置 | 观测位置 | 真实误差（未知） |
|------|------|----------|------------------|
| A | (116.0, 39.0) | (116.5, 39.5) | az=+1.0°, r=+100m |
| B | (117.0, 39.5) | (116.5, 39.5) | az=-0.5°, r=-50m |
| C | (118.0, 40.0) | (116.5, 39.5) | az=+0.3°, r=+80m |

**优化过程**：
```
初始状态: 所有误差 = 0
         修正后点分散，空间方差 = 1500m
         代价 J = 100×1500 + 0 + 0 + 0 = 150000

迭代1: 尝试 A.az += 0.1°
        方差 → 1450m, J = 145005 ↓ ✓ 接受

迭代2: 尝试 B.az += 0.1°
        方差 → 1480m, J = 148005 ↑ ✗ 拒绝
        尝试 B.az -= 0.1°
        方差 → 1420m, J = 142005 ↓ ✓ 接受

...继续迭代...

最终结果:
  A.az = +1.0°, A.r = +100m
  B.az = -0.5°, B.r = -50m
  C.az = +0.3°, C.r = +80m
  空间方差 = 50m
  代价 J = 100×50 + 0.15×0.54 + 6e-7×7700 + 0 ≈ 5000
```

## 九、算法优势与局限

### 9.1 优势

1. **无需外部参考**：利用多雷达观测一致性，不需要外部真值
2. **分步优化降低复杂度**：坐标下降法简化高维优化问题
3. **多步长策略**：快速收敛到较优解
4. **代价函数约束**：避免极端解，提高结果可靠性

### 9.2 局限

1. **依赖观测一致性**：需要足够的匹配组数量
2. **局部最优**：梯度下降可能陷入局部最优
3. **计算复杂度**：匹配组数量增加时计算量增大
4. **误差耦合**：方位角和距离误差可能存在一定耦合
