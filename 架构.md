# RFTIP 项目架构文档

## 1. 项目概述

RFTIP (RadarFusionTrack Intelligence Platform) 是一个雷达轨迹监测与智能分析平台，用于管理和分析飞行轨迹数据，支持实时可视化、禁飞区管理和 AI 智能分析功能。

## 6. 数据流

### 6.1 用户认证流程

1. 用户注册 -> 发送验证码 -> 验证通过 -> 创建用户
2. 用户登录 -> 验证用户名密码 -> 生成 JWT Token -> 返回 Token
3. 前端存储 Token -> 后续请求携带 Token -> 后端验证 Token

### 6.2 文件上传流程

1. 前端选择文件 -> 切片上传到后端
2. 后端接收文件 -> 存储到 MinIO -> 保存元信息到 MySQL
3. 返回文件 ID -> 前端显示上传成功

### 6.3 轨迹分析流程

1. 用户上传轨迹文件 -> 后端解析 -> 存储原始数据
2. 轨迹预处理 -> 卡尔曼滤波 -> 去噪处理 -> 存储处理后数据
3. AI 分析 -> 异常检测 -> 风险评估 -> 返回分析结果
4. 前端 3D 可视化 -> 展示轨迹点和飞行路径

## 9. 开发规范

### 9.1 后端规范

- 使用 Pydantic 进行数据验证
- 遵循 RESTful API 设计原则
- 分层架构：Router -> Service -> Model
- 使用 async/await 异步编程

### 9.2 前端规范

- 使用 TypeScript 确保类型安全
- 使用 Pinia 进行状态管理
- 使用 apiCall 包装器统一处理 API 错误
- 组件化开发，遵循 Vue 3 Composition API 规范
