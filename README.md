# 基于多源数据融合的雷达轨迹监测与智能分析平台

**RadarFusionTrack Intelligence Platform (RFTIP)**

## 1. 项目背景与目标

在现代化空域管理中，雷达观测数据常受环境干扰和设备精度影响产生噪声。本项目旨在构建一个集**轨迹去噪优化**、**自适应误差校准**、**地理信息增强**以及**大模型意图识别**于一体的 Web 可视化平台，解决原始雷达数据"看得见但看不准、看不懂"的问题。

---

## 2. 技术栈架构

| 维度 | 技术选型 | 说明 |
| --- | --- | --- |
| **前端框架** | Vue 3 + Vite + TypeScript | 高性能响应式界面开发 |
| **三维引擎** | **Cesium.js** | 实现全球尺度下的飞机轨迹、高度、地理围栏可视化 |
| **后端框架** | FastAPI (Python) | 异步高性能架构，原生支持 Python 科学计算库 |
| **数据库** | MySQL 8.0 | 存储雷达站元数据、飞行批号统计及 AI 分析结果 |
| **文件存储** | MinIO (或本地 Static 存储) | 兼容 S3 协议，存储原始大规模轨迹 CSV/Excel 文件 |
| **大模型** | DeepSeek / GPT-4o / Ollama | 结合空间上下文进行飞行意图推理 |
| **核心算法** | Kalman Filter, RANSAC, Geopy | 负责轨迹平滑、误差剔除及逆地理编码 |

## 3. 核心功能模块设计

### 3.1 轨迹处理与场景自适应算法

系统支持用户根据实际工况预设两种分析场景：

* **场景一：多源参考模式（大部分雷达可靠）**
* **原理：** 当多台雷达探测同一目标时，利用 **RANSAC (随机抽样一致性)** 算法剔除偏离群体的"坏点"站号，并计算该站的系统性偏差。
* **目的：** 自动识别并校准故障/低精度雷达。


* **场景二：单源盲测模式（不确定可靠性）**
* **原理：** 采用 **卡尔曼滤波 (Kalman Filter)**。基于物理运动模型（匀速/匀加速）对单站噪声数据进行预测与修正。
* **目的：** 获得平滑连续的飞行轨迹，消除雷达跳变点。


### 3.2 地理增强 AI 分析 (MCP 模式)

系统不直接将坐标点扔给 AI，而是通过"空间特征提取层"进行增强：

1. **特征提取：** 自动识别轨迹中的起飞地、降落地、转弯点、盘旋区。
2. **逆地理编码：** 结合 **MCP (Model Context Protocol)** 思路，调用地理数据库将经纬度转化为"北京市某禁飞区边缘"、"某领海交界线"等语义文本。
3. **智能推理：** AI 结合空间语义与飞行特征（高度、速度、轨迹形状）生成分析报告。

### 3.3 三维数字孪生可视化

* **多维轨迹对比：** 动态同屏对比"原始噪声轨迹（红线）"与"算法优化轨迹（绿线）"。
* **高度感应墙：** 轨迹线下方的垂直投影墙，直观展示飞机高度变化。
* **地理围栏告警：** 预设敏感区域，当飞机侵入时自动改变轨迹颜色并弹窗提醒。

## 4. 数据库逻辑设计 (核心表)

```sql
-- 1. 雷达站基础信息表
CREATE TABLE radar_stations (
    station_id VARCHAR(50) PRIMARY KEY, -- 站号
    longitude DECIMAL(10, 7),
    latitude DECIMAL(10, 7),
    altitude FLOAT,                     -- 雷达站自身高度
    description VARCHAR(255)
);

-- 2. 飞行批号汇总表
CREATE TABLE flight_batches (
    batch_id VARCHAR(50) PRIMARY KEY,   -- 飞机批号
    start_time DATETIME,
    end_time DATETIME,
    max_altitude FLOAT,
    avg_speed FLOAT,
    ai_analysis_report TEXT,            -- 大模型生成的意图报告
    risk_level ENUM('low', 'mid', 'high')
);

-- 3. 轨迹坐标明细表 (与文件存储互补)
CREATE TABLE track_details (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    batch_id VARCHAR(50),
    station_id VARCHAR(50),
    time_stamp DATETIME(6),
    raw_lon DECIMAL(10, 7),             -- 原始经度
    raw_lat DECIMAL(10, 7),             -- 原始纬度
    opt_lon DECIMAL(10, 7),             -- 优化经度
    opt_lat DECIMAL(10, 7)              -- 优化纬度
);

```

## 5. 业务流程图

1. **数据载入：** 用户上传 CSV -> 后端存入 MinIO -> 解析入库。
2. **配置选择：** 用户在 Web 端选择"算法策略"（融合校准/单站平滑）。
3. **计算处理：** 后端执行 Python 算法逻辑 -> 更新数据库中的 `opt_` 坐标字段。
4. **AI 增强：** 提取关键坐标 -> 逆地理查询地理名称 -> 组装 Prompt -> 调用大模型。
5. **可视化展示：** 前端 Cesium 加载两类轨迹，ECharts 渲染高度/误差曲线，AI 板块展示分析结论。

## 6. 项目创新点总结

* **算法灵活性：** 区分了"已知基准"与"未知基准"两种工程实际场景。
* **感知深度：** 突破了单纯的数字展示，通过地理语义增强让 AI 理解"飞机在哪里、在干什么"。
* **交互专业性：** 结合 Cesium 三维引擎，提供达到行业演示标准的 4D（空间+时间）可视化效果。
